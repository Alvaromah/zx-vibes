HELLO_ WORLD

Tu primer programa en una máquina nueva, con Claude Code

Partiendo solo del paquete público @zx-vibes/toolkit, instalas Spectral desde npm, lo conectas a Claude Code por MCP y le pides que imprima HELLO WORLD en un Spectrum 48K emulado — viendo la pantalla del resultado. Todos los pasos están verificados; el ensamblador del final se construyó y ejecutó con el toolkit antes de escribir esto. 

# requisitos mínimos
node --version      # >= 20
claude --version    # Claude Code

# backend por defecto opcional
sjasmplus --version # si usas sjasmplus

# y un único install
npm install -g @zx-vibes/toolkit

¿Por qué sin el repo?

Publicar en npm es justo lo que desbloquea este flujo: el toolkit (zxs + zxs-mcp) y la ROM del 48K viajan dentro del paquete. No clonas nada.

El backend por defecto usa sjasmplus; zxs doctor te dice si falta y cómo instalarlo. Para evitar ese binario, usa el backend embebido spectral en el flujo starter/recetas.

⓪ Requisitos

Node, Claude Code y un backend de ensamblador. La ROM no se descarga aparte — va dentro del paquete (vía zx-generation).

Node.js ≥ 20

node --version

Backend de ensamblador

Por defecto: sjasmplus en el PATH. Sin binario externo: ZXS_ASSEMBLER=spectral para starters/recetas.

Claude Code

claude --version

# si usas sjasmplus en macOS: se compila desde fuente
tar xf sjasmplus-*-src.tar.xz && cd sjasmplus-*
make && sudo make install

①→② Instala y verifica

Un único install global. Te deja zxs y zxs-mcp en el PATH:

npm install -g @zx-vibes/toolkit

¿Sin global? npx -p @zx-vibes/toolkit zxs <cmd> también vale.

El doctor confirma el toolchain:

zxs doctor

✓ node: v22.x.x
✓ sjasmplus: v1.23.1
✓ @zx-vibes/asm: available
✓ 48k.rom: .../zx-generation/rom/48k.rom

Si sjasmplus sale en rojo, instálalo o ejecuta ZXS_ASSEMBLER=spectral zxs doctor para la ruta embebida.

③ Conecta Spectral a Claude Code (MCP)

# una carpeta de trabajo y registras el MCP
mkdir hello-zx && cd hello-zx
claude mcp add spectral -- zxs-mcp

# -s user lo deja en TODOS tus proyectos
claude mcp list   # comprobar

👁️ Claude ve la pantalla

La tool zx_screen devuelve el display como imagen, más un OCR del texto en rejilla 32×24. Por eso el agente puede comprobar visualmente lo que imprime, no solo leer un JSON.

Otras tools: zx_build, zx_run, zx_inspect, zx_debug, zx_keys, zx_state.

④ Pídeselo al agente

Lanza claude en la carpeta y dale el encargo. Encadenará zx_build → zx_run → zx_screen solo.

prompt >

Crea un programa para ZX Spectrum 48K que imprima HELLO WORLD en pantalla usando la rutina de impresión de la ROM. Ensámblalo con la tool de Spectral, ejecútalo y enséñame la pantalla.

resultado → la rejilla de zx_screen muestra HELLO WORLD arriba y © 1982 Sinclair Research Ltd al pie (la línea de arranque)

Alternativa sin agente — la CLI a mano

Si prefieres ver el bucle sin el agente, crea hello.asm (verificado en el emulador) y constrúyelo tú.

; hello.asm — sin codigos AT
    ORG 0x8000
start:
    ld a, 2         ; canal 2 = pantalla
    call 0x1601    ; ROM CHAN-OPEN
    ld hl, msg
print_loop:
    ld a, (hl)
    or a            ; 0 termina la cadena
    jr z, done
    rst 0x10       ; imprime A
    inc hl
    jr print_loop
done:
    jr done         ; bucle: no vuelvas a la ROM
msg:
    defb "HELLO WORLD", 0
zxs build hello.asm
zxs run --bin build/hello.bin \
    --org 0x8000 --frames 50 \
    --no-detect-hangs \
    --screenshot screen.png --text

⚠️ Dos trampas

El bucle infinito final es obligatorio: con un ret el control vuelve a la ROM, BASIC repinta y borra tu texto.

Por eso --no-detect-hangs: el bucle es intencional, pero el watchdog lo marcaría como tight-loop (salida 2). La captura se hace igual al terminar.

La trampa clásica que esta guía evita a propósito: imprimir con AT y,x dentro de una cadena terminada en cero — si y o x son 0, el 0 se confunde con el terminador y la cadena se trunca. Documentado en docs/reference/common-bugs.md.
