¿Ensamblador o BASIC?

Ensamblador Z80. El Spectrum traía BASIC, pero es interpretado y lento: mueve decenas de operaciones por fotograma y un juego de acción necesita miles. Los juegos comerciales de la época eran todos ensamblador, y Spectral está construido para eso: backend embebido @zx-vibes/asm por defecto para starters/recetas, sjasmplus opcional para compatibilidad avanzada, depurador, recetas y watchdog hablan Z80.

¿Nunca has escrito ensamblador? Para eso es este tutorial — y la trampa final del paso ⑨: pedírselo al agente.

⓪→① El proyecto y el bucle sagrado

Requisitos y proyecto nuevo:

# una vez
git clone https://github.com/Alvaromah/spectral
npm install && npm run build
zxs doctor   # ¿Node, ROM y backend ok?

# tu juego
zxs new arkanoid && cd arkanoid

zxs new crea un juego que ya funciona: src/main.asm, lib/ con rutinas probadas, tests/, y el manual AGENTS.md/CLAUDE.md por si lo pilota un agente.

El gesto que repetirás cien veces:

zxs build src/main.asm                 # 1. compila
zxs run --bin build/main.bin \
        --org 0x8000 --frames 300     # 2. ejecuta
zxs screen --text                     # 3. MIRA

El JSON de zxs run dice si el programa está sano (status: "ok", haltSynced: true); zxs screen enseña la pantalla. Nunca asumas que algo funciona sin mirar.

② El escenario: marcador + pala

La pantalla se direcciona por celdas de 8×8 píxeles (32×24). cell_addr convierte (fila, columna) en dirección de memoria. Las variables del juego viven en direcciones fijas (PADX equ 0xBF00) — así los tests pueden leerlas con zxs mem read. Y ei + halt es el latido: 50 pulsos por segundo.

; fill_cell — el byte E en las 8 lineas de (B=fila, C=col)
fill_cell:
    push bc
    call cell_addr
    ld b, 8
.lines:
    ld (hl), e
    inc h          ; siguiente linea de la celda
    djnz .lines
    pop bc
    ret

⚠️ El bug clásico (nos mordió escribiendo esto)

Imprimir el HUD con el código AT dentro de la cadena — db 22, 0, 0, "SCORE…" — no imprime nada: el impresor termina en el byte 0, y AT 0,0 lleva dos ceros como operandos. El mismo bug que cometió el agente del Pong.

La cura: un helper print_at que manda el AT por registros (ld a,22 / rst 0x10 / ld a,fila / rst 0x10 / ld a,col / rst 0x10) — inmune al terminador.

verifica → zxs screen --text debe enseñar SCORE 000000 · LIVES 3 arriba y ███ en la fila 22

③ Mover la pala

read_qaop (de la lib) devuelve el teclado en bits: O = izquierda, P = derecha. La regla de oro del movimiento sin estelas:

borra en la posición vieja → actualiza la variable → pinta en la nueva

¿Probar teclado sin teclado? Teclas programadas: --keys pulsa por ti en fotogramas exactos. Como el emulador es determinista, el resultado es idéntico siempre — jugar se convierte en testear.

zxs run --bin build/main.bin --org 0x8000 \
        --frames 120 --fresh \
        --keys "10:P*40,70:O*20"

# P 40 frames (pala al tope, col 29),
# O 20 frames (retrocede 20)
# → acaba EXACTAMENTE en la columna 9:
'         ███                    '

④ La bola: rebotes, vidas y game over

sprite XOR

Dibujar la bola dos veces en el mismo sitio la borra (receta 04). Movimiento limpio gratis: la misma rutina dibuja y borra.

celdas + dirección ±1

La bola vive en coordenadas de celda con (dx, dy) = ±1 y se mueve cada 4 fotogramas. Rebotar = negar dx o dy. Retro masticable y suficiente.

el orden de cada paso

borrar → calcular rebotes (pared, techo, pala, suelo) → actualizar → pintar. La fila 22 es la pala; tocar la 23 = vida perdida.

verifica → 1000 frames sin teclas pierden las 3 vidas: "GAME OVER - SPACE" · y --keys "5:SPACE*10" reinicia la partida

Detalle de oficio: ld sp, 0xBF00 como primera instrucción de start — cada reinicio llega vía jp start sin limpiar la pila; con pila propia no crece partida a partida.

⑤ Ladrillos, puntuación y victoria

La pantalla es el modelo de datos: no hay array de ladrillos — un ladrillo existe si su celda tiene píxeles. La colisión lee un byte de la celda destino; si no es cero, hay ladrillo: borrarlo, sumar 10 (BCD, receta 10), invertir dy.

El muro: 5 filas × 28 columnas, un color por fila vía atributos (el byte de color de cada celda: papel*8 + tinta).

El determinismo brilla: la bola sirve hacia arriba y golpea su primer ladrillo siempre en la fila 7, columna 19, hacia el fotograma 20 → SCORE 000010.

🔬 El truco del depurador

¿Cómo pruebas la pantalla de victoria sin una partida perfecta de 140 ladrillos? Escribiendo en la memoria viva:

zxs mem write 0xBF07 01  # BRICKS = 1
zxs run --frames 60
zxs screen --text        # → YOU WIN! - SPACE

El emulador es tuyo: tócalo por dentro.

⑥ Tests automáticos

// tests/arkanoid.test.json
{
  "build": "../src/main.asm",
  "frames": 60,
  "assert": [
    { "type": "status",  "equals": "ok" },
    { "type": "haltSynced", "equals": true },
    { "type": "screenIncludes", "text": "SCORE 000010" },
    { "type": "pixelAt", "x": 156, "y": 59, "set": false },
    { "type": "pixelAt", "x": 148, "y": 59, "set": true },
    { "type": "pixelAt", "x": 120, "y": 180, "set": true }
  ]
}

Léelo en voz alta: a los 60 fotogramas el marcador dice 000010, el píxel del ladrillo destruido está apagado, su vecino encendido, y la pala donde debe. Solo es posible porque mismo arranque + mismas teclas ⇒ ejecución idéntica, siempre.

⑦ A la galería · ⑧ Tus deberes

Publicar en la galería

zxs run ... --screenshot screen.png
zxs state export --z80 game.z80

Crea gallery/games/<tu-id>/ con el snapshot, la captura, un meta.json (prompt, modelo, esfuerzo, controles) y un transcript.md. El contrato es la transparencia: sin procedencia no hay juego.

Curiosidad: si el borde sale amarillo en el navegador, es un bug del cargador .z80 de zx-generation 1.0.1, ya arreglado upstream — pendiente de release.

La siguiente iteración (ejercicios)

  1. Física de pala — el tercio izquierdo/derecho devuelve la bola con dx contrario
  2. Sonido — receta 09: beep al romper, zap al morir
  3. Niveles — al ganar, otro muro y más velocidad
  4. Bola por píxeles — receta 03: pasos de 4 px, colisiones más finas

⑨ La otra forma: pídeselo al agente

Todo lo que acabas de hacer a mano, un agente lo hace solo — esa es la tesis de Spectral. En el directorio del proyecto, abre Claude Code y di:

Build a playable Arkanoid. Follow CLAUDE.md.

Compilará, ejecutará, mirará la pantalla, se equivocará, leerá los veredictos del watchdog y se corregirá. El Pong de la galería salió exactamente así: ~8 ciclos, 40 herramientas, 11 minutos, cero ayuda.
