① La idea en una frase

Spectral es una caja de herramientas para que una IA escriba juegos de ZX Spectrum ella sola — los compile, los ejecute, vea la pantalla, encuentre sus propios errores y los corrija, igual que lo haría una persona delante del ordenador.

Y la segunda capa que lo hace especial: el emulador sobre el que corre todo, zx-generation, lo generó también una IA (es tuyo, está en tu GitHub). El resultado:

emulador escrito por IA × juegos escritos por IA × herramientas escritas por IA

② El vocabulario

Las diez palabras que lo explican todo. Con esto, el resto se lee solo.

ZX Spectrum 48K

Ordenador británico de 1982. Pantalla 256×192, 48 KB de memoria, procesador Z80. Los juegos se escriben en ensamblador: instrucciones directas al procesador.

Emulador

Programa moderno que imita ese hardware antiguo. zx-generation es el nuestro: un ZX Spectrum que vive dentro de JavaScript.

Headless

"Sin cabeza": ejecutar el emulador sin ventana, solo en memoria. Corre 132× más rápido que el hardware real — una IA puede hacer miles de pruebas por minuto.

@zx-vibes/asm

El ensamblador embebido: traduce el código fuente Z80 (texto) a binario sin instalar programas externos. sjasmplus queda como backend opcional.

zxs (la CLI)

Nuestra caja de herramientas de comandos: zxs build compila, zxs run ejecuta, zxs screen enseña la pantalla. Es lo que usa el agente en su bucle.

MCP

El protocolo con el que Claude se conecta a herramientas. Nuestro servidor zxs-mcp le da un Spectrum vivo: ejecuta código y ve la pantalla como imagen en la conversación.

Watchdog

El "perro guardián": clasifica cuelgues. No dice "no funciona" — dice por qué: "HALT con interrupciones apagadas", "bucle infinito", "pila corrupta"…

Git · GitHub · repo

Git guarda la historia del código en commits (instantáneas con autor y mensaje). GitHub la publica. Un repo público lo puede leer y usar cualquiera.

CI (Integración Continua)

Un robot de GitHub que, en cada cambio, compila y ejecuta todos los tests en máquinas limpias (Linux, macOS…). Si algo se rompe, se pone rojo y te enteras al momento.

PR (Pull Request)

La forma civilizada de proponer un cambio: "aquí está mi mejora, revísala y si te gusta, incorpórala (merge)".

④ La sesión de hoy, paso a paso

4.1 El punto ciego del watchdog

Cuando el agente construía el Pong, un error estrelló el programa contra el editor de BASIC (la pantalla de inicio del Spectrum). El watchdog no lo detectó. ¿Por qué? El editor es un programa educado: duerme la CPU entre fotogramas, exactamente igual que un juego sano. Un coche parado en la cuneta con el motor al ralentí parece un coche aparcado.

El arreglo: el watchdog ahora vigila desde dónde se ejecuta el código. Tu juego vive en la RAM; el editor vive en la ROM. Si un programa que corría en RAM lleva más de un segundo ejecutando solo ROM → veredicto pc-in-rom: "tu programa ha perdido el control y ha caído al sistema". Con explicación y sugerencias, como todos los veredictos.

4.2 El libro de recetas, completo (12/12)

Una receta es un bloque de código Z80 documentado, copiable, con test automático — si está en el libro, funciona, y el CI lo garantiza. Hoy se escribieron las 6 que faltaban:

sprites 16×16 enmascarados — muñecos sin borrar el fondo
IM2 — código que corre 50 veces/segundo, el ritual completo
efectos de sonido — pitidos sin estropear el borde
marcador BCD — puntuaciones sin divisiones
azar (PRNG) — aleatorio reproducible, clave para tests
efectos de color — animación barata con atributos

Detalle bonito: la receta del azar se verificó dos veces — el Z80 en el emulador y una simulación independiente en JavaScript produjeron exactamente los mismos bytes.

4.3 La galería web

La meta comunitaria: una galería de juegos hechos por IA, jugables en el navegador, con transparencia total. Hoy existe. Cada juego es una ficha: captura, modelo de IA, el prompt, iteraciones, tiempo, controles, procedencia… y el botón ▶ Jugar.

El truco de "Jugar": al Pong corriendo en el emulador le hicimos una foto congelada de toda su memoria (un snapshot .z80). La página carga zx-generation —tu emulador también corre en navegador— y le inyecta esa foto. El juego revive en el punto exacto donde estaba.

Verificado con un Chrome automatizado: la página renderiza y el Pong corre.

4.4 La publicación en GitHub

Decidiste: repo público y commits con tu email noreply de GitHub (enlaza los commits a tu cuenta sin exponer tu correo personal). Se reescribió la historia completa con esa identidad antes de publicar — después ya no se puede.

El robot de CI se estrenó… y cazó un bug real al primer intento: la regla build/ del .gitignore (pensada para ignorar binarios compilados) también se tragaba las carpetas de código fuente llamadas src/build/. El primer push subió el proyecto sin el módulo que habla con el ensamblador. En tu máquina todo funcionaba; en la máquina limpia del CI, no. Para esto exactamente existe el CI. Arreglado y verificado.

Y se activó GitHub Pages (hosting gratuito que sirve ficheros del repo): la galería quedó online.

4.5 Cuatro regalos a zx-generation

Construyendo Spectral encontramos cuatro problemas en tu emulador. Se convirtieron en cuatro PRs, cada uno con tests nuevos:

  • #2El cargador de snapshots leía mal el color del borde — el formato .z80 lo guarda en los bits 1-3 de un byte; el código leía los bits 0-2. Nos afecta directo: la galería carga snapshots.
  • #3Guardar estado perdía los "registros sombra" — el Z80 tiene registros de repuesto que casi todos los juegos usan; la función de guardado los olvidaba. Como una partida guardada que pierde la mitad del inventario.
  • #4El emulador no arrancaba fuera del navegador — pedía cosas que solo existen en navegadores (canvas, document) aunque no fuera a usarlas. Ahora corre headless en Node, que es lo que Spectral hace a todas horas.
  • #5Una puerta de entrada oficial — antes había que importar las piezas internas "por la ventana" (rutas de ficheros internos); ahora hay un índice público y un mapa de exports. Verificado que las rutas viejas siguen funcionando.

4.6 La "arqueología": un CI que nunca estuvo verde

Al abrir los PRs, sus checks salieron rojos. Investigando apareció la sorpresa: no eran los cambios — el robot de CI de zx-generation llevaba roto desde siempre. Nunca había pasado en verde. Capas, como una excavación:

capa 1 — GitHub retiró una pieza de los workflows: todo run moría al arrancar, antes de un solo test
capa 2 — el job de calidad comprobaba un fichero con nombre inexistente y leía una salida que jest nunca imprime
capa 3 — exigía 80% de cobertura de tests; la real es ~47%. Un listón decorativo = rojo eterno
capa 4 — 27 ficheros fuera del formato del propio repo, 13 errores de lint, avisos de seguridad en dependencias

Con tu autorización se mergearon los 4 PRs (+ uno de seguridad pendiente desde julio) y un PR de reparación integral: acciones actualizadas, pasos arreglados, formato y lint saldados, auditoría limpia, y el listón de cobertura puesto en el valor real como trinquete — ya no falla siempre; falla solo si alguien la empeora.

Resultado: el primer CI verde de la historia de zx-generation. Y de propina, su documentación quedó publicada online.

⑥ Lo que falta (y quién hace cada cosa)

1 · Publicar el paquete en npm (te toca a ti + yo)

npm es la tienda de paquetes de JavaScript: publicar @zx-vibes/toolkit permite a cualquiera usar npx zxs sin clonar nada. Solo necesito que escribas ! npm login en la sesión; del resto (publicar v0.1.0 y verificar instalación limpia) me encargo yo.

2 · Versión nueva de zx-generation (te toca a ti + yo)

Las mejoras están en main pero sin publicar. Una release se dispara etiquetando: git tag v1.1.0 && git push --tags (sugiero 1.1.0, no 1.0.2: hay funcionalidad nueva, no solo arreglos). Cuando exista, yo actualizo Spectral para usarla y quito los apaños internos.

3 · Grabar el vídeo (te toca a ti, guion listo)

Con el runbook en la mano: terminal grande, asciinema o QuickTime, el prompt mínimo, y a dejar que el agente trabaje. Yo te acompaño en el re-run cuando quieras.
